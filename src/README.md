# Request Processor Microservice

Микросервис для приема запросов на отправку уведомлений с гарантированной доставкой с использованием паттерна **Transactional Outbox**.

## Технологии

* Java 21
* Spring Boot 3.2.1
* Apache Kafka
* PostgreSQL
* Flyway
* Docker

## Эндпоинты

### POST /api/v1/notifications

Создание нового уведомления.

#### Тело запроса

```json
{
  "type": "SMS | EMAIL | PUSH | TG_MESSAGE",
  "message": "текст сообщения"
}
```

#### Ответ

```json
{
  "id": "uuid",
  "type": "SMS",
  "message": "текст сообщения",
  "status": "ACCEPTED"
}
```

## Описание

Сервис принимает запрос на отправку уведомления и сохраняет его в базе данных вместе с записью в **outbox-таблице** в рамках одной транзакции.

Отдельный процесс считывает события из outbox и публикует их в **Apache Kafka**, что обеспечивает надежную доставку сообщений и предотвращает потерю данных при сбоях.

## Статусы уведомлений

* **ACCEPTED** — уведомление принято системой
* **SENT** — уведомление успешно отправлено
* **FAILED** — произошла ошибка при отправке
